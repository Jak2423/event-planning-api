3.2 Системийн ерөнхий бүтэц

«Nairly» арга хэмжээний төлөвлөлтийн систем нь клиент–сервер архитектурт суурилсан, гурван үндсэн давхаргаас бүрдэнэ. Эхний давхарт хэрэглэгчийн интерфэйсийг хариуцсан веб программууд, хоёр дахь давхарт бизнес логик болон REST API сервер, гурав дахь давхарт өгөгдлийн сан, нэвтрэлт, файлыг хадгалах үйлчилгээ байрлана. Клиент программууд сервертэй HTTP протоколоор холбогдож, Supabase-ийн JWT токен ашиглан баталгаажуулалт хийнэ.

Зураг 3.1. Nairly системийн ерөнхий бүтэц

Клиент давхарга. «event-planning» нь үйлчлүүлэгч болон үйлчилгээ үзүүлэгчийн үйл ажиллагааг нэгтгэсэн Next.js веб програм бөгөөд байршил хайх, захиалга өгөх, арга хэмжээний төлөвлөгөө бүтээх зэрэг функцуудыг хангана. «event-planning-admin» нь администраторын самбар бөгөөд захиалга, үйлчилгээ үзүүлэгчийг удирдах зориулалттай. Хоёр програм хоёулаа тусдаа deploy хийгдэж, нэг REST API серверт хандаж ажиллана.

Серверийн давхарга. «event-planning-api» нь Hono фреймворк дээр бичигдсэн Node.js сервер бөгөөд бүх бизнес логик, үнийн тооцоолол, өгөгдлийн баталгаажуулалтыг гүйцэтгэнэ. Middleware давхарга JWT токенийг шалгаж, хэрэглэгчийн эрхийг (customer, provider, admin) тодорхойлно. Routes модулиуд HTTP хүсэлтийг хүлээн авч, холбогдох функц руу чиглүүлнэ.

Өгөгдлийн давхарга. Supabase платформ PostgreSQL өгөгдлийн сан, хэрэглэгчийн нэвтрэлт (Auth), зураг хадгалах Storage үйлчилгээг нэг дор өгнө. Сервер service-role түлхүүр ашиглан өгөгдөлд хандаж, хандалтын хяналтыг програмын түвшинд хэрэгжүүлнэ.
