3.3 Домэйн загварын класс диаграм

Дараах зурагт «Nairly» системийн үндсэн домэйн объектууд (entity), тэдгээрийн атрибутууд болон хоорондын харилцааг UML класс диаграмын хэлбэрээр илэрхийлэв. Систем нь хэрэглэгч, байршил, үйлчилгээ, төлөвлөгөө, захиалга гэсэн гол классуудаас бүрдэнэ.

Зураг 3.2. Nairly системийн домэйн класс диаграм

User — Supabase Auth хэрэглэгч OrderItem — orders.items JSON дотор хадгалагдах value object ServiceOption.priceAdjustment — сонголтын бүрэн үнэ

User класс нь системийн бүх оролцогчийг илэрхийлж, customer, provider, admin гэсэн эрхийн түвшинг агуулна. Venue болон ProviderService классууд provider-т хамаарна. VenueEventPackage нь байршлын арга хэмжээний багцыг, VenuePackageService нь багцад багтах үйлчилгээний мөрийг тодорхойлно. ProviderService-д ServiceOptionGroup болон ServiceOption композит харилцаагаар сонголтын боломжуудыг нэмнэ. EventPlan нь хэрэглэгчийн төсөв, сонгосон байршил болон үйлчилгээнүүдийг нэгтгэнэ; Order нь баталгаажсан захиалгыг OrderItem объектуудын JSON массиваар хадгална.
